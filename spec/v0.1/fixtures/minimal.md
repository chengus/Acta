# Minimal Acta v0.1 fixture

This document annotates every byte in [`minimal.acta`](minimal.acta), the
deterministic compatibility fixture generated by
[`format_probe.py`](../format_probe.py). The file contains one schema with one
non-nullable UTC microsecond timestamp column and one data block with three
rows.

All multi-byte integers are little-endian. Offsets below are absolute file
offsets unless a field explicitly says it is relative to a frame header or
payload.

Logical contents:

| Row | `time` (`timestamp64[us, UTC]`) | UTC instant |
| ---: | ---: | --- |
| 0 | 1,000,000 | `1970-01-01T00:00:01Z` |
| 1 | 2,000,000 | `1970-01-01T00:00:02Z` |
| 2 | 3,000,000 | `1970-01-01T00:00:03Z` |

## Layout

### File prologue: 0x000–0x03f

| Offset | Bytes | Meaning |
| --- | --- | --- |
| `000–007` | `4143 5441 0d0a 1a0a` | File magic: `ACTA\r\n\x1a\n` |
| `008–009` | `0000` | Format major: `0` |
| `00a–00b` | `0100` | Format minor: `1` |
| `00c–00f` | `4000 0000` | Prologue size: `0x40 = 64` |
| `010–017` | all zero | Feature flags: row IDs disabled |
| `018–027` | `00 01 … 0f` | 16-byte file ID |
| `028–02f` | `4000 ...` | Schema frame starts at offset `64` |
| `030–03b` | all zero | Reserved |
| `03c–03f` | `0070 430b` | Prologue CRC32C |

Because feature flags are zero, blocks do not receive implicit row IDs.

## Reader walkthrough

1. Read and validate the 64-byte prologue, including its CRC and schema offset.
2. At `0x040`, validate the schema-frame prefix before trusting its lengths.
3. Read the schema header and descriptor, then validate the schema trailer.
4. Advance by the schema frame's total length to `0x0d0`.
5. Validate the data-frame prefix and use its header and payload lengths to
   locate the trailer without searching for magic bytes.
6. Read the block header. Use its timestamp bounds for range pruning and its
   table offsets to locate the column and stream descriptors.
7. For a projection containing `time`, read the stream's 24 stored bytes,
   validate their CRC32C, and decode three little-endian `int64` values.
8. Validate the commit trailer. Strict validation also checks the frame body
   CRC32C.
9. Advance by 264 bytes to `0x1d8`, where this fixture reaches EOF.

## Interrupted append and corruption behavior

If EOF occurs anywhere inside the data frame before its complete trailer is
available, a normal growing-file reader keeps `0x0d0` as its retry offset and
continues exposing the already committed schema. A recovery open reports
`0x0d0` as `last_good_offset` and ignores the interrupted tail.

A checksum failure in a complete final frame may be discarded only as an
explicit repair decision. A checksum failure before a later valid frame is
mid-file corruption and must be reported. CRC32C detects damage but cannot
reconstruct lost values.

## Regenerating and validating

From the repository root:

```bash
uv run spec/v0/format_probe.py \
  --write-fixture spec/v0/fixtures/minimal.acta
shasum -a 256 spec/v0/fixtures/minimal.acta
```

Expected SHA-256:

```text
1e40278e2f58a597faef56107bd6a31048cabf5b5173619f8112ad787c6f658a
```

The generator validates the fixture, every possible truncation point inside
the final frame, and representative corruption in the prologue, frame prefix,
header, payload, and trailer. See the complete normative definitions in
[`format_v0.md`](../format_v0.md).
